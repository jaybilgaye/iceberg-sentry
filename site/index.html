before it costs you a query.

Iceberg Sentry is a high-performance, stateless linter for Apache Iceberg. It scores table health, flags delete-file amplification, finds orphan storage, samples for PII, and gates CI — all from metadata alone. No Spark cluster required.

Get started → $ curl -sSL raw.githubusercontent.com/jaybilgaye/iceberg-sentry/main/scripts/install.sh

Built for "Day 2" operations

Iceberg solved ACID on object storage. Nobody built the tooling to keep tables healthy over time. Iceberg Sentry is that tooling — a single static binary you can run from a laptop, a CI job, or a Kubernetes CronJob.

// detect

Delete-file amplification

First-class diagnostic. Counts position vs. equality deletes, computes per-partition ratio, predicts read amplification before queries get slow.

// detect

Partition skew

Coefficient-of-variation analysis across partitions. Surfaces hot keys and sparse partitions with concrete re-partitioning recommendations.

// detect

Orphan files

Bloom-filter-backed differential scan between metadata and storage. Dry-run by default. Grace period excludes in-flight writes.

// detect

Untagged PII

Streams Parquet row groups, runs regex + Shannon-entropy detection for emails, Luhn-valid cards, SSNs, API keys. Zero-disk persistence.

// classify

Write-pattern aware

Streaming, batch, or mixed — inferred from the snapshot history. Streaming tables get more lenient file-size thresholds automatically.

// integrate

CI-native exit codes

Exit codes 0–5 mapped to OK / warn / critical / untagged-PII / config error / connection failure. Wire into any pipeline.

// integrate

SARIF for GitHub Code Scanning

--format sarif uploads natively to the GitHub Security tab. Each failing health dimension is a SARIF result with `level=error|warning`.

// integrate

Prometheus exporter

One-shot push to a pushgateway, or a long-lived /metrics server with built-in ServiceMonitor + Grafana dashboard JSON.

// migrate

HDFS → CDP Readiness Audit

Pre-flight scan for on-prem-to-cloud migrations. Flags absolute hdfs:// paths, HDFS-specific properties, v1 tables. Per-table LOW/MEDIUM/HIGH risk score.

Closes the loop with sentry bench

Wire it into anything

// github actions

SARIF + PR comment

- name: Iceberg health gate
  run: |
    iceberg-sentry audit \
      --catalog glue --namespace finance \
      --format sarif > iceberg.sarif

- uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: iceberg.sarif
    category: iceberg-health
// kubernetes

Helm chart, CronJob + exporter

helm install sentry deploy/helm \
  --set cronjob.enabled=true \
  --set exporter.enabled=true \
  --set serviceMonitor.enabled=true
