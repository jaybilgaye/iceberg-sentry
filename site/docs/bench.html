bench

Capture a labelled health snapshot, run your maintenance, re-scan, and print the delta. Turns a diagnostic into a number you can put in a PR description.

Subcommands

$ iceberg-sentry bench start    # capture
$ iceberg-sentry bench compare  # re-scan + diff

Workflow

# 1. Capture pre-maintenance baseline
$ iceberg-sentry bench start \
    --catalog rest --rest http://catalog:8181 \
    --table finance.transactions --tag pre-compact

# 2. Run your compaction (Spark, CDE job, …)
$ spark-sql -e "CALL catalog.system.rewrite_data_files('finance.transactions')"

# 3. Compare
$ iceberg-sentry bench compare \
    --catalog rest --rest http://catalog:8181 \
    --table finance.transactions --tag pre-compact
~ bench compare

Flags (shared by both subcommands)

FlagDefaultDescription
--tablerequiredFully qualified table id.
--tagdefaultLabel for this baseline (e.g. pre-compact).
--bench-dir.sentry/benchWhere baselines are persisted on disk.

Plus all catalog and storage flags from audit.

In a CI workflow

- name: Capture baseline
  run: iceberg-sentry bench start --table finance.transactions --tag pr-{{ github.event.number }}

- name: Apply Spark migration
  run: ./scripts/run-compaction.sh

- name: Diff and comment on PR
  run: |
    iceberg-sentry bench compare --table finance.transactions \
        --tag pr-{{ github.event.number }} \
      | gh pr comment ${{ github.event.number }} --body-file -
