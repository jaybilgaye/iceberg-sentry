Quickstart

Generate a small Iceberg warehouse with pyiceberg, run your first audit, and learn the basics — in under 5 minutes.

1. Generate sample tables

We ship a Python script that uses pyiceberg to write real Iceberg v2 tables (metadata.json + Avro manifests + Parquet) to a local directory. These are the exact same bytes a production engine would produce.

$ pip install "pyiceberg[pyarrow,sql-sqlite]>=0.7.0"
$ python scripts/gen_fixtures.py --root ./warehouse --clean
Wrote fixtures under /tmp/warehouse

This creates five tables under the fixtures namespace:

TableShape
fixtures.healthy_v2Single large file, no deletes.
fixtures.streaming_v250 small commits; classified as a streaming write pattern.
fixtures.delete_amp_v2v2 table with multiple delete passes.
fixtures.skewed_v2Day-partitioned; one hot partition with 100k rows, six sparse ones with 100 each.
fixtures.pii_v25,000 rows of email + Luhn-valid card data plus a few high-entropy API keys.

2. Audit a single table

$ iceberg-sentry audit \
    --catalog localfs --catalog-root ./warehouse \
    --table fixtures.skewed_v2
~ audit fixtures.skewed_v2 
  Table: fixtures.skewed_v2  │  Score: 68/90  CRITICAL
  ─────────────────────────────────────────────────────────────────
  [CRITICAL]   file_size               8/20  100% of files below 32MB
                  → rewrite_data_files (target=512MB) recommended
  [OK]         delete_amplification   20/20  no delete files present
  [OK]         manifest_density       15/15  7 manifest files
  [OK]         snapshot               15/15  7 snapshots; oldest 0m
  [CRITICAL]   partition_skew          5/15  CV=30.12 across 7 partitions (1 hot, 0 sparse)
                  → re-partition hot keys or bucket the partition column
  [OK]         format_version          5/5   Iceberg v2
  ─────────────────────────────────────────────────────────────────
  Catalog: localfs:./warehouse  │  Snapshot: 8263…  │  Format: v2  │  Scan: 9ms

Sentry detected the skew on the day-partitioned table (CV = 30, one hot partition).

3. Get JSON for downstream tools

$ iceberg-sentry audit --catalog localfs --catalog-root ./warehouse \
    --table fixtures.skewed_v2 --format json | jq '{table, score, write_pattern, worst_severity}'
{
  "table": "fixtures.skewed_v2",
  "score": 68,
  "write_pattern": "streaming",
  "worst_severity": "CRITICAL"
}

4. Scan a whole namespace

$ iceberg-sentry audit --catalog localfs --catalog-root ./warehouse \
    --namespace fixtures --format text | head -20

The exit code is the worst severity across all tables. Use --fail-on warn to escalate warnings to failures.

5. Apply a policy file

$ cat sentry.yaml
version: "1.0"
default_catalog: localfs
policies:
  - name: fixtures
    target_namespace: "fixtures.*"
    min_file_size_mb: 128
    max_manifest_files: 500
    delete_file_ratio_fail: 0.25
    min_health_score: 75

$ iceberg-sentry audit --catalog localfs --catalog-root ./warehouse \
    --namespace fixtures --policy sentry.yaml --fail-on warn

6. Find orphan files

$ # plant a backdated orphan
$ echo "junk" > ./warehouse/fixtures/healthy_v2/data/orphan-zzz.parquet
$ touch -d "3 days ago" ./warehouse/fixtures/healthy_v2/data/orphan-zzz.parquet

$ iceberg-sentry orphans --catalog localfs --catalog-root ./warehouse \
    --table fixtures.healthy_v2 --grace-period 24h
~ orphans
  [DRY RUN] Orphan File Report: fixtures.healthy_v2
  ─────────────────────────────────────────────────────────────────
  Scanned prefix:  warehouse/fixtures/healthy_v2/data
  Snapshot:        602432555078556252 (locked)
  Grace period:    24h0m0s
  Active files:    4
  Scanned objects: 2

  Found 1 orphan candidate(s)  │  Reclaimable: 5 B

  Preview:
    warehouse/fixtures/healthy_v2/data/orphan-zzz.parquet  (5 B)
// safety Iceberg Sentry never deletes files. orphans always emits a recommendation; you wire up your preferred deletion tool against the JSON manifest.

7. Scan for PII

$ iceberg-sentry pii --catalog localfs --catalog-root ./warehouse \
    --table fixtures.pii_v2 \
    --row-group-fraction 1.0 \
    --rows-per-group 5000
~ pii
  PII Scan Report: fixtures.pii_v2
  ─────────────────────────────────────────────────────────────────
  Files sampled: 1
  Findings:      3 (review required — no auto-tagging)
