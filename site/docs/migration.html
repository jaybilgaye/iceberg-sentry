migration

On-prem HDFS → CDP Public Cloud Migration Readiness Audit. Pure metadata work — runs against your Hive Metastore, no compute required.

What it checks

  1. Absolute hdfs:// paths anywhere in the metadata (location, manifest-list URI, data-file paths sampled from the current snapshot's manifests).
  2. HDFS-specific table properties that don't honour on S3FileIO/AzureFileIO — e.g. write.metadata.path, write.distribution-mode=hadoop.
  3. v1 format tables: cheaper to upgrade to v2 on-prem with a warm cache than after migration over S3.
  4. Total reachable bytes — informs your S3/ADLS landing cost estimate.

Risk scoring

HIGH≥1 HIGH finding (HDFS path detected).
MEDIUM≥1 MEDIUM finding (HDFS property, v1 table).
LOWClean.

Example

$ iceberg-sentry migration \
    --catalog hive --hive hms.onprem:9083 \
    --namespace finance --format json | jq
{
  "table": "finance.transactions",
  "risk": "HIGH",
  "findings": [
    {
      "code": "HDFS_PATH",
      "severity": "HIGH",
      "message": "absolute hdfs:// path in metadata: hdfs://nn/user/hive/warehouse/...",
      "remediation": "re-register the table with a location-relative metadata path before migration"
    },
    {
      "code": "HDFS_PROPERTY",
      "severity": "MEDIUM",
      "message": "table property write.metadata.path=/tmp/hms-meta — absolute HDFS path; rewrite to a location-relative path",
      "remediation": "ALTER TABLE finance.transactions UNSET TBLPROPERTIES ('write.metadata.path')"
    }
  ],
  "total_data_bytes": 1843298423,
  "manifest_count": 84,
  "data_file_count": 1228
}

Flags

FlagDescription
--table / --namespaceSingle table or scan the entire namespace.
--formattext · json

All catalog/storage flags from audit.

Exit codes

  • 2 — at least one HIGH-risk table found.
  • 1 — at least one MEDIUM-risk table.
  • 0 — clean.
