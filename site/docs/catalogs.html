Catalogs

Sentry talks to the catalog you already operate to resolve namespace.table → metadata location. The actual table bytes are read directly from storage.

LocalFS

Directory-backed catalog matching HadoopCatalog's layout:

<root>/<namespace>/<table>/metadata/vN.metadata.json
                          /data/...
$ iceberg-sentry audit \
    --catalog localfs --catalog-root ./warehouse \
    --table finance.transactions

Useful for tests, air-gapped clusters, and the pyiceberg-generated fixtures shipped in scripts/gen_fixtures.py.

AWS Glue Data Catalog

Reads the metadata_location table parameter (Iceberg's Glue convention).

$ iceberg-sentry audit \
    --catalog glue --namespace finance

Cross-account access — pass --glue-catalog <account-id> and assume the target role via your AWS profile or environment.

$ AWS_PROFILE=glue-reader iceberg-sentry audit \
    --catalog glue --glue-catalog 123456789012 \
    --namespace finance

Hive Metastore (Thrift)

Speaks the HMS Thrift protocol directly via the apache/thrift Go runtime. We implement just the two RPCs we need (get_table, get_all_tables); no thrift-generated bloat.

$ iceberg-sentry audit \
    --catalog hive --hive hms.example.com:9083 \
    --namespace finance

Kerberos

For Cloudera CDP deployments running hive.metastore.sasl.enabled=true:

$ iceberg-sentry audit \
    --catalog hive --hive hms.example.com:9083 \
    --hive-principal "iceberg-sentry@EXAMPLE.COM" \
    --hive-keytab /etc/iceberg-sentry/sentry.keytab \
    --namespace finance

The keytab is loaded via gokrb5; /etc/krb5.conf is consulted automatically (override with KRB5_CONFIG).

Iceberg REST (Polaris, Tabular, Unity, Nessie)

Conforms to the Iceberg REST spec. Aliases polaris and unity are accepted in addition to rest.

Bearer token

$ iceberg-sentry audit \
    --catalog rest --rest https://catalog.example.com/api/catalog \
    --rest-token "$SENTRY_REST_TOKEN" \
    --namespace finance

OAuth2 client credentials (Polaris / Unity)

$ iceberg-sentry audit \
    --catalog polaris --rest https://polaris.example.com/api/catalog \
    --rest-oauth-client-id "$CLIENT_ID" \
    --rest-oauth-client-secret "$CLIENT_SECRET" \
    --rest-oauth-token-url https://polaris.example.com/api/oauth/token \
    --namespace finance

The adapter caches the access token and refreshes it 30s before expires_in.

Storage backends

The storage layer is dispatched by URI scheme returned by the catalog:

SchemeBackendNotes
file:// / bare pathLocal FSFor dev + LocalFS catalog.
s3://aws-sdk-go-v2 S3Works against MinIO/LocalStack with --s3-path-style.
hdfs://WebHDFSPass the WebHDFS root via --hdfs URL.

ADLS Gen2 and GCS are accessed via their S3-compatible endpoints today; native backends are on the Phase 4 roadmap.
