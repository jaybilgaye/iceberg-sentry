Deployment

Kubernetes CronJob, long-lived Prometheus exporter, Helm chart, Grafana dashboard. Everything you need to run Sentry in production.

Periodic CronJob

Push-gateway delivery. Use this when you want fire-and-forget scans on a schedule.

apiVersion: batch/v1
kind: CronJob
metadata:
  name: iceberg-sentry-daily-audit
  namespace: data-platform
spec:
  schedule: "0 2 * * *"
  concurrencyPolicy: Forbid
  jobTemplate:
    spec:
      template:
        spec:
          restartPolicy: Never
          containers:
            - name: sentry
              image: ghcr.io/jaybilgaye/iceberg-sentry:latest
              args:
                - audit
                - --catalog=glue
                - --namespace=finance
                - --policy=/config/sentry.yaml
                - --format=prometheus
                - --push-gateway=http://prometheus-pushgateway:9091
              envFrom:
                - secretRef: { name: iceberg-sentry-aws }
              volumeMounts:
                - { name: policy, mountPath: /config, readOnly: true }
              securityContext:
                runAsNonRoot: true
                readOnlyRootFilesystem: true
                allowPrivilegeEscalation: false
                capabilities: { drop: ["ALL"] }
          volumes:
            - name: policy
              configMap: { name: iceberg-sentry-policy }

Long-lived exporter

Scraped by Prometheus / kube-prometheus-stack ServiceMonitor.

apiVersion: apps/v1
kind: Deployment
metadata: { name: iceberg-sentry-exporter, namespace: data-platform }
spec:
  replicas: 1
  selector: { matchLabels: { app.kubernetes.io/name: iceberg-sentry } }
  template:
    metadata: { labels: { app.kubernetes.io/name: iceberg-sentry } }
    spec:
      containers:
        - name: sentry
          image: ghcr.io/jaybilgaye/iceberg-sentry:latest
          args: [ export, --catalog=glue, --namespace=finance, --listen=:9400, --interval=5m ]
          ports: [{ name: metrics, containerPort: 9400 }]
          readinessProbe:
            httpGet: { path: /healthz, port: 9400 }
          envFrom:
            - secretRef: { name: iceberg-sentry-aws }
---
apiVersion: v1
kind: Service
metadata: { name: iceberg-sentry-exporter, namespace: data-platform }
spec:
  selector: { app.kubernetes.io/name: iceberg-sentry }
  ports: [{ name: metrics, port: 9400, targetPort: metrics }]
---
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata: { name: iceberg-sentry, namespace: data-platform }
spec:
  selector: { matchLabels: { app.kubernetes.io/name: iceberg-sentry } }
  endpoints: [{ port: metrics, interval: 60s }]

Helm chart

The repo ships deploy/helm/ with the chart. CronJob and exporter are independently toggleable.

$ helm install sentry deploy/helm \
    --set cronjob.enabled=true \
    --set exporter.enabled=true \
    --set serviceMonitor.enabled=true \
    --set image.tag=latest

Pod security defaults

The shipped manifests harden:

  • runAsNonRoot: true, UID 65534
  • readOnlyRootFilesystem: true
  • allowPrivilegeEscalation: false
  • capabilities.drop: ["ALL"]
