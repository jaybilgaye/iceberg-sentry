export

Long-lived Prometheus exporter. Serves /metrics and /healthz, re-runs the configured audit in the background on an interval.

When to use this vs. push gateway

  • Push gateway — iceberg-sentry audit --format prometheus --push-gateway URL from a Kubernetes CronJob. Fire-and-forget.
  • Long-lived exporter — iceberg-sentry export in a Deployment. Scraped by Prometheus / kube-prometheus-stack ServiceMonitor.

Example

$ iceberg-sentry export \
    --catalog glue --namespace finance \
    --listen :9400 --interval 5m
iceberg-sentry export: listening on :9400 (scan interval=5m0s)

Endpoints

GET /metricsPrometheus exposition format with a leading # scanned_at <rfc3339> comment.
GET /healthz200 if the last scan succeeded, 503 otherwise.

Flags

FlagDefaultDescription
--listen:9400Listen address.
--interval5mBackground re-scan cadence.
--namespace / --tableTargets to scan (mutually exclusive).
--branchmainIceberg branch.

All catalog and storage flags from audit apply.

Metric shape

# HELP iceberg_table_health_score Composite Iceberg Sentry health score (0..max).
# TYPE iceberg_table_health_score gauge
iceberg_table_health_score{catalog="rest:http://catalog",format_version="2",table="finance.transactions",write_pattern="streaming"} 78

# HELP iceberg_table_dimension_score Per-dimension health score.
# TYPE iceberg_table_dimension_score gauge
iceberg_table_dimension_score{...,dimension="file_size"} 8
iceberg_table_dimension_score{...,dimension="delete_amplification"} 20
...

# HELP iceberg_table_dimension_severity 0=OK,1=INFO,2=WARNING,3=CRITICAL
# TYPE iceberg_table_dimension_severity gauge
iceberg_table_dimension_severity{...,dimension="file_size"} 3

Grafana dashboard

Ready-to-import JSON ships at deploy/grafana-dashboard.json. Panels:

  • Health Score by Table (table view, thresholded red/orange/green)
  • Critical Findings Over Time (timeseries, severity == 3)
  • Reclaimable Storage (stat, summed wastage bytes)
