Iceberg Sentry

Iceberg-native lakehouse reliability linter — diagnostics, CI gating, and operational insight for Apache Iceberg tables, from metadata alone. No Spark cluster required.

// what it does Reads Iceberg vN.metadata.json + Avro manifest files directly. Computes a 0–100 health score across six dimensions, finds orphan storage, samples for PII, classifies write patterns, and emits structured output for CI gating and metrics scraping.

Catalogs & storage

Pick the catalog you already operate; Sentry resolves the table metadata location through it.

CatalogAuthFlag
LocalFS (HadoopCatalog-style)none--catalog localfs --catalog-root
AWS Glue Data CatalogIAM--catalog glue [--glue-catalog]
Hive Metastoreplain or Kerberos--catalog hive --hive HOST:PORT
Iceberg REST · Polaris · Unitybearer / OAuth2--catalog rest --rest URL

Storage backends are picked from the URI scheme returned by the catalog: s3://, hdfs:// (WebHDFS), or local paths.

Exit codes

Designed to gate CI pipelines.

0OK — all checks passed
1Warning threshold exceeded (use --fail-on warn to surface)
2Critical — one or more dimensions failed
3Untagged PII detected
4Tool / config error
5Catalog or storage connection failure
