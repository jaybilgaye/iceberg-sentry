CI Integration

Iceberg Sentry is built shift-left: structured output, deterministic exit codes, SARIF for native GitHub integration. Wire it into any CI.

GitHub Actions

SARIF upload to Code Scanning

name: Iceberg health
on: [push, pull_request, schedule]
permissions:
  contents: read
  security-events: write
jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Sentry
        run: curl -sSL https://raw.githubusercontent.com/jaybilgaye/iceberg-sentry/main/scripts/install.sh | bash

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.ICEBERG_AUDIT_ROLE }}
          aws-region: us-east-1

      - name: Run audit
        run: |
          iceberg-sentry audit \
            --catalog glue --namespace finance \
            --policy sentry.yaml \
            --format sarif > iceberg.sarif

      - uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: iceberg.sarif
          category: iceberg-health

Results appear in Security → Code scanning alerts grouped by dimension.

Pull-request comment

- name: Comment on PR
  if: github.event_name == 'pull_request'
  run: |
    iceberg-sentry audit \
      --catalog glue --namespace finance \
      --format json > report.json
    jq -r '"## Iceberg Health\n- Table: `\(.table)`\n- Score: \(.score)/\(.max_score) (\(.worst_severity))"' \
      report.json | gh pr comment ${{ github.event.number }} --body-file -
  env:
    GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

Bench-driven PR comment for compactions

- name: Capture baseline
  run: iceberg-sentry bench start --table finance.transactions --tag pr-${{ github.event.number }}

- run: ./scripts/run-spark-compaction.sh

- name: Diff and comment
  run: |
    iceberg-sentry bench compare --table finance.transactions \
        --tag pr-${{ github.event.number }} \
      | gh pr comment ${{ github.event.number }} --body-file -

GitLab CI

iceberg-audit:
  image: ghcr.io/jaybilgaye/iceberg-sentry:latest
  stage: test
  script:
    - iceberg-sentry audit
        --catalog rest --rest $REST_URL
        --namespace finance
        --policy sentry.yaml
        --format sarif > iceberg.sarif
  artifacts:
    reports:
      sast: iceberg.sarif
    when: always

Pre-commit hook

# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: iceberg-sentry
        name: Iceberg Table Health Check
        entry: iceberg-sentry audit --policy sentry.yaml --namespace staging --fail-on warn
        language: system
        pass_filenames: false

Exit code reference

0OK — pass
1WARNING surfaced (only with --fail-on warn)
2CRITICAL — hard fail
3Untagged PII detected (pii command)
4Tool / config error
5Catalog / storage connection failure

Output formats

textDefault. Colour-aware terminal output.
jsonOne JSON object per table. Pipe through jq.
sarifSARIF 2.1.0 for GitHub / GitLab Code Scanning ingest.
prometheusExposition format for push gateway / scraping.

Perf regression gate

The repo ships .github/workflows/perf-regression.yml which runs Go benchmarks for the scan path, compares to the target branch with benchstat, fails the PR on > 20% regression, and posts the diff as a PR comment.
