pii

Stream-sample Parquet row groups for emails, Luhn-validated credit cards, SSNs, phone numbers, AU TFNs, IN Aadhaars, and high-entropy API keys. Zero-disk persistence.

// design intent

The scanner is built so column values never escape the byte stream. We pipe Parquet values directly into the regex/entropy classifier and discard them immediately. Only column names and detected PII types appear in the output. The CLI never logs a sampled value.

How it works

  1. Resolve the table; pick the current snapshot.
  2. Walk manifests, gather up to --max-files Parquet URIs.
  3. For each file, open it via the storage backend, sample --row-group-fraction of row groups, and read up to --rows-per-group rows from each.
  4. For every string column value, run regex patterns (with Luhn validation for cards) and Shannon-entropy detection (for API keys).
  5. Aggregate hits per (column, PII type). Surface findings whose hit rate exceeds --min-confidence.

Examples

Defaults (5% row-group sample)

$ iceberg-sentry pii \
    --catalog glue --table finance.customers

Exhaustive scan of a small table + Atlas payload

$ iceberg-sentry pii \
    --catalog localfs --catalog-root ./warehouse \
    --table fixtures.pii_v2 \
    --row-group-fraction 1.0 --rows-per-group 5000 \
    --min-confidence 0.5 \
    --atlas-output atlas-pii-$(date +%F).json

Flags

FlagTypeDefaultDescription
--tablestringrequiredFully qualified table id.
--row-group-fractionfloat0.05Probability that a row group is read (0..1).
--rows-per-groupint1024Max rows scanned from each sampled group.
--min-confidencefloat0.5Hit rate threshold to surface a finding.
--max-filesint16Cap on the number of Parquet files sampled per table.
--atlas-outputpathWrite Atlas/UC bulk-import payload. - for stdout.
--formatstringtexttext · json · atlas

Detectors

PII typeHow
EMAILRFC-ish regex.
SSN_USNNN-NN-NNNN regex.
CREDIT_CARD13–19 digit run + Luhn validation to filter false positives.
PHONE_E164+<digits> with 7–14 trailing digits.
AU_TFN9-digit Australian Tax File Number pattern.
IN_AADHAAR12-digit Aadhaar pattern. Suppressed when a credit-card surface is present to avoid double-counting.
API_KEY_ENTROPYShannon entropy ≥ 4.0 bits over 24+ alphanumeric chars, no whitespace.

Atlas / Unity Catalog payload

{
  "review_required": true,
  "findings": [
    {
      "table": "fixtures.pii_v2",
      "column": "email",
      "pii_type": "EMAIL",
      "sample_count": 5000,
      "match_count": 5000,
      "confidence": 1.0,
      "recommended_tag": "PII_EMAIL"
    }
  ]
}

The payload is shaped for Atlas's POST /api/atlas/v2/entity/bulk after operator review; no auto-tagging.

Exit code

Exits 3 (untagged PII) when at least one finding clears the confidence threshold; 0 otherwise.
