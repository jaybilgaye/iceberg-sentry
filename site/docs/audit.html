audit

Score Iceberg table health across six dimensions. The primary command — and the one you'll wire into CI.

Usage

$ iceberg-sentry audit [flags]

You must specify either --table or --namespace.

Examples

Single table from a local catalog

$ iceberg-sentry audit \
    --catalog localfs --catalog-root ./warehouse \
    --table finance.transactions

Whole namespace from AWS Glue, fail CI on warnings

$ iceberg-sentry audit \
    --catalog glue --namespace finance \
    --policy sentry.yaml --fail-on warn

SARIF for GitHub Code Scanning

$ iceberg-sentry audit \
    --catalog rest --rest https://polaris.example.com/api/catalog \
    --namespace finance --format sarif > iceberg.sarif

Prometheus push to a CronJob's pushgateway

$ iceberg-sentry audit \
    --catalog glue --namespace finance \
    --format prometheus \
    --push-gateway http://prometheus-pushgateway:9091 \
    --push-job iceberg-sentry

Audit a specific branch

$ iceberg-sentry audit --table finance.transactions --branch staging

SARIF output

One result per failing dimension. The logicalLocations entry carries the table id; properties exposes score and max-score so dashboards can chart trends across runs.

Prometheus output

Emits six gauge families:

  • iceberg_table_health_score
  • iceberg_table_health_max_score
  • iceberg_table_dimension_score{dimension="…"}
  • iceberg_table_dimension_max_score{dimension="…"}
  • iceberg_table_dimension_severity (0=OK, 1=INFO, 2=WARNING, 3=CRITICAL)
  • iceberg_table_scan_duration_ms

All series carry table, catalog, format_version labels; branch and write_pattern when set.

Exit codes

See Concepts → Exit codes. --fail-on warn escalates WARNING into a non-zero exit; --fail-on never always exits 0 (useful for metrics-only CronJobs).
